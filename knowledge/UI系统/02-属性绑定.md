# Y3 UI 属性绑定系统

Y3 UI 支持把引擎数据直接绑定到控件属性，用于自动刷新生命、魔法、玩家资源、进度条等界面内容。

常规脚本里优先使用以下公开接口：

- `ui:bind_player_prop(uiAttr, player, attr_or_var, accuracy)`
- `ui:bind_unit_attr(uiAttr, attr_name, accuracy)`
- `ui:bind_global_variable(uiAttr, globalVar, accuracy)`
- `ui:bind_unit(unit)`

对于经验值、等级、波次、自定义统计等非标准展示，仍然建议手动更新 UI。

---

## 绑定方式对比

| 方式 | 适用场景 | 数据来源 | 是否自动刷新 |
|------|----------|----------|--------------|
| `bind_player_prop` | 玩家资源或其他玩家属性展示 | `Player` | 是 |
| `bind_unit_attr` | 单位属性，如生命、魔法、物理攻击 | 默认主控/高亮单位；调用 `bind_unit` 后改为指定单位 | 是 |
| `bind_global_variable` | 绑定编辑器/ECA 触发系统全局变量 | 编辑器全局变量名 | 是 |
| 手动更新 | 经验、等级、波次、自定义字符串 | 自定义逻辑 | 否 |

## 1. 玩家属性绑定：`bind_player_prop`

### 接口特性

`bind_player_prop` 在 y3 lualib 中确实存在。

不过这里要区分两层：

- 运行时 lualib 里的 [maps/EntryMap/script/y3/game/const.lua](maps/EntryMap/script/y3/game/const.lua) 当前把 `y3.const.PlayerAttr` 定义为空表。
- VSCode扩展 y3-helper 会在保存地图时自动更新 [maps/EntryMap/script/y3-helper/meta/playerAttrs.lua](maps/EntryMap/script/y3-helper/meta/playerAttrs.lua)，给 `y3.const.PlayerAttr` 补充作者自定义的玩家属性枚举。

这意味着：

- Lua 层公开接口存在。
- 在本项目里，`y3.const.PlayerAttr` 可能会因为 y3helper 自动生成的 meta 而出现项目自定义键，例如 `gold`。

### 代码示例

如果你的项目已经确认某个玩家资源键可用，可以这样绑定：

```lua
local goldText = y3.ui.get_ui(player, "ResourceHUD.gold_TEXT")
if goldText then
    goldText:bind_player_prop('文本', player, 'gold', 0)
end
```

上面的 `gold` 属于项目或引擎侧约定的资源键，不是 `y3.const.PlayerAttr` 中的公开常量。

在本项目里，更准确地说，它会被 y3-helper 自动写入 `y3-helper/meta/playerAttrs.lua`，因此可以作为项目级补全枚举使用，但它不是 `y3/game/const.lua` 里原生预置的那一批常量。

### 参数说明

| 参数 | 说明 |
|------|------|
| `uiAttr` | UI 属性。公开枚举见下文，只能稳定写 `'文本'`、`'当前值'`、`'最大值'` |
| `player` | 数据来源玩家 |
| `attr_or_var` | 玩家属性键，会透传到底层 |
| `accuracy` | 小数精度，默认 `0` |

## 2. 单位属性绑定：`bind_unit_attr`

### 公开枚举与透传字符串

`UI:bind_unit_attr` 的 Lua 实现会先查 `y3.const.UnitAttr[attr_name]`，查不到时再把原字符串直接透传到底层。

因此这里要区分两类写法：

- 公开枚举：在 `y3.const.UnitAttr` 里明确定义。
- 透传字符串：demo 或项目里能用，但不属于 Lua 层公开枚举。

### 常见公开 `UnitAttr`

- `'生命'`
- `'魔法'`
- `'最大生命'`
- `'最大魔法'`
- `'攻击速度'`
- `'物理攻击'`
- `'物理防御'`

### 常见透传字符串

官方 demo 里还直接传了这些字符串：

- `'力量'`
- `'敏捷'`
- `'智力'`

它们出现在 demo 代码里，但不在 `y3.const.UnitAttr` 的公开枚举表中。

### 直接对 UI 控件绑定

`UI:bind_unit_attr` 的公开签名是：

```lua
ui:bind_unit_attr(uiAttr, attr_name, accuracy)
```

如果不额外调用 `bind_unit(unit)`，编辑器客户端默认会跟随当前主控/高亮单位更新，也就是当前选择链路里的 `highlight_unit`。

如果你希望控件稳定跟随某个明确单位，而不是随着玩家当前选中对象变化，就要再调用一次 `bind_unit(unit)` 覆盖这个默认来源。

如果该控件需要跟随某个具体单位刷新，再调用 `bind_unit(unit)` 指定单位来源：

```lua
local hpText = y3.ui.get_ui(player, "HUD.hp_TEXT")
if hpText then
    hpText:bind_unit_attr('文本', '生命', 0)
    hpText:bind_unit(hero)
end
```

### 在 LocalUILogic 中绑定子控件

如果你使用的是本地 UI 逻辑层，接口形态是：

```lua
logic:bind_unit_attr(child_name, uiAttr, unit_attr, accuracy)
```

示例：

```lua
MAIN:bind_unit_attr('血条.进度条', '当前值', '生命')
MAIN:bind_unit_attr('血条.进度条', '最大值', '最大生命')
```

### 不建议直接写成绑定的属性

- `'当前生命'`
- `'当前魔法'`
- `'当前经验'`
- `'等级'`

这些名字没有在 `y3.const.UnitAttr` 中作为公开枚举出现。

其中经验和等级在官方 demo 中都是手动更新：

- 经验：`unit:get_exp()`、`unit:get_upgrade_exp()`
- 等级：`unit:get_level()`

```lua
expText:set_text(tostring(hero:get_exp()))
levelText:set_text(string.format('等级%d', hero:get_level()))
```

## 3. UI 属性 `uiAttr`

`bind_unit_attr`、`bind_player_prop`、`bind_global_variable` 这几个接口都会用到 `uiAttr`。

Lua 层公开的 `y3.const.UIAttr` 只有这三个值：

- `'文本'`
- `'当前值'`
- `'最大值'`

因此不要把它写成下面这些不存在于公开枚举里的说法：

- `'进度'`
- `'进度最大值'`
- `'图片'`

图片更新应使用 `set_image()` 一类接口，而不是走这组属性绑定接口。

## 4. 全局变量绑定：`bind_global_variable`

这个接口绑定的是编辑器 / ECA 触发系统里的全局变量，不是 Lua 环境里的 `_G.xxx`。

也就是说：

- `bind_global_variable('文本', 'SomeVar', 0)` 读取的是触发系统变量名 `SomeVar`
- 它不会自动读取 Lua 里的 `_G.SomeVar`
- 如果你的数据本来只存在于 Lua 模块、`_G`、或某个脚本状态表里，通常不适合用这个接口

因此在纯 Lua 逻辑里，这个接口通常用不上，更推荐自行实现刷新逻辑，例如通过 `set_text()`、事件回调或定时器更新。

```lua
local waveText = y3.ui.get_ui(player, "ResourceHUD.wave_TEXT")
if waveText then
    waveText:bind_global_variable('文本', 'ResourceHUD_currentWave', 0)
end
```

如果波次数据只是 Lua 侧状态，例如：

```lua
_G.ResourceHUD_currentWave = 3
waveText:set_text('第' .. tostring(_G.ResourceHUD_currentWave) .. ' 波')
```

那通常直接手动更新更合适，不必强行走 `bind_global_variable`。

## 5. 手动更新模式

### 适用场景

- 经验值：`unit:get_exp()`、`unit:get_upgrade_exp()`
- 等级：`unit:get_level()`
- 波次信息
- 拼接字符串后的描述文本
- 任何不在公开枚举内的数据

### 代码示例

```lua
y3.ltimer.loop(0.5, function()
    local expText = y3.ui.get_ui(player, "ResourceHUD.exp_TEXT")
    if expText and hero then
        expText:set_text('经验: ' .. tostring(hero:get_exp()))
    end
end)

function updateWaveDisplay(waveNum)
    local waveText = y3.ui.get_ui(player, "ResourceHUD.wave_TEXT")
    if waveText then
        waveText:set_text('第' .. tostring(waveNum) .. ' 波')
    end
end
```

## 6. `bind_unit` 的使用限制

`bind_unit(unit)` 的作用是覆盖控件默认的主控/高亮单位来源，把属性绑定切到你显式指定的单位上。它本身不会凭空生成绑定规则，所以不能对一个完全未配置绑定属性的控件直接生效。

### 什么时候能用

以下两种情况至少满足一种：

| 前置条件 | 说明 |
|----------|------|
| 已调用 `bind_unit_attr` | 代码里已经为该控件声明了要绑定哪个单位属性 |
| 编辑器里已有绑定配置 | 导出的 JSON 中存在 `text_bind`、`current_value_bind`、`max_value_bind` 等字段 |

### 已配置绑定的控件示例

```json
{
    "name": "label_1",
    "type": 3,
    "text_bind": {
        "__tuple__": true,
        "items": [3, "official_res_1"]
    },
    "uid": "xxx"
}
```

这里的 `text_bind` 与 Lua 层 `y3.const.UIAttr['文本']` 对应；`official_res_1` 是项目数据中的资源键，不是 y3 lualib 的公开常量。

### 未配置绑定的控件示例

```json
{
    "name": "label_1_1",
    "type": 3,
    "uid": "xxx"
}
```

### 正确用法

```lua
local label = y3.ui.get_ui(player, "panel.label")
label:bind_unit_attr('文本', '生命', 0)
label:bind_unit(unit)
```

如果该控件的绑定规则已经在编辑器中配置完成，也可以只在运行时切换单位：

```lua
local label = y3.ui.get_ui(player, "panel.label")
label:bind_unit(unit)
```

### 常见错误

```lua
local label = y3.ui.get_ui(player, "panel.label")
label:bind_unit(unit)
```

如果控件没有任何绑定规则，这样调用后通常不会得到期望结果。

## 常见错误

### 错误 1：把 `bind_unit_attr` 写成错误参数形态

```lua
-- 错误：把 unit 当成第二个参数
hpText:bind_unit_attr('文本', hero, y3.const.UnitAttr['生命'])
```

`UI` 控件的公开接口并不接收 `unit` 作为第二个参数。应先声明绑定属性，再通过 `bind_unit(unit)` 指定单位。

### 错误 2：把非公开枚举名写成公开 `UnitAttr`

```lua
expText:bind_unit_attr('文本', '当前经验')
levelText:bind_unit_attr('文本', '等级')
```

`当前经验`、`等级` 都不在公开的 `y3.const.UnitAttr` 枚举中。经验和等级更稳妥的方式是手动更新。

### 错误 3：把 `UI` 当成 `LocalUILogic`

```lua
local waveText = y3.ui.get_ui(player, "ResourceHUD.wave_TEXT")

waveText:on_refresh(function()
    return _G.currentWave
end)
```

`on_refresh` 属于 `LocalUILogic`，不是 `UI` 对象的方法。这也不等价于 UI 属性绑定。对于波次、拼接文案、自定义状态，更直接的方式通常是主动 `set_text()`。

### 错误 4：跨模块状态不同步

```lua
local currentWave = 1
```

如果多个模块都要读取该值，应统一走 `_G`、状态模块或其他明确的共享层。

---

## 最佳实践

1. `uiAttr` 只按公开枚举写 `'文本'`、`'当前值'`、`'最大值'`。
2. 单位基础属性优先用 `bind_unit_attr`，并明确单位来源。
3. 经验、等级、波次、自定义描述，优先直接手动更新。
4. 对透传字符串保持谨慎，不要把 demo 能用的名字都写成“公开枚举”。
5. 编辑器里已经配置好 `text_bind` 一类字段的控件，运行时只做数据源切换，不重复造一套绑定逻辑。
6. `bind_global_variable` 更偏向编辑器 / ECA 触发系统变量；纯 Lua 状态展示通常直接自己刷新更稳妥。
