# Y3 编辑器 UI 适配规则

## 概述

UI 适配不是单一功能，而是三层规则共同作用的结果：

1. 画布分辨率如何随游戏窗口变化。
2. 控件坐标如何相对父节点计算。
3. 控件是否开启基于父节点边缘的适配。

这三层逻辑叠加后，才决定一个控件在不同分辨率、不同父容器尺寸下的最终位置和大小。

---

## 坐标系统

先区分两类名字：

- `pos_data`、`open_adapter`、`adapter_option`、`size` 等是**编辑器 JSON 字段**，用于描述界面资源数据。
- `ui:set_pos(...)`、`ui:set_ui_size(...)`、`ui:set_text_alignment(...)` 等才是 **y3 Lua API**。

本文会同时说明这两层概念，但两者不要混为同一类“接口”。

Y3 UI 画布原点在左下角，而不是常见的左上角。

```text
Y轴 ↑
    │
1080├─────────────────────┐
    │                     │
    │      画布区域        │
    │                     │
   0└─────────────────────┴──→ X轴
    0
```

- X 轴从左往右增加。
- Y 轴从下往上增加。
- 坐标既可以用像素，也可以用百分比表示。

### pos_data 字段

```json
"pos_data": {
    "__tuple__": true,
    "items": [pos_x, pos_y, percent_x, percent_y, x_mode, y_mode]
}
```

| 字段 | 说明 | 示例 |
|------|------|------|
| `pos_x` | 设计画布中的 X 像素坐标 | `100.0` |
| `pos_y` | 设计画布中的 Y 像素坐标 | `50.0` |
| `percent_x` | 相对父节点宽度的百分比，范围 `0-100` | `50.0` 表示水平居中 |
| `percent_y` | 相对父节点高度的百分比，范围 `0-100` | `50.0` 表示垂直居中 |
| `x_mode` | X 坐标模式，`0=像素`、`1=百分比` | `0` |
| `y_mode` | Y 坐标模式，`0=像素`、`1=百分比` | `0` |

以下内容来自**编辑器侧数据定义**，不是 `maps/EntryMap/script/y3` 内的 Lua API。

根据编辑器源码 `clients/ui/ui_const.py`：

- `UI_POS_DATA_ABS_DEF = (None, None, 50, 50, 0, 0)`
- `UI_POS_DATA_PCT_DEF = (0, 0, 50, 50, 1, 1)`

这说明 `pos_data` 后两位是坐标模式位，不是对齐枚举。

### 常见位置示例

假设根画布为 `1920x1080`：

| 位置 | pos_x | pos_y | percent_x | percent_y |
|------|-------|-------|-----------|-----------|
| 左下角 | 0 | 0 | 0 | 0 |
| 右下角 | 1920 | 0 | 100 | 0 |
| 左上角 | 0 | 1080 | 0 | 100 |
| 右上角 | 1920 | 1080 | 100 | 100 |
| 屏幕中央 | 960 | 540 | 50 | 50 |
| 顶部中央 | 960 | 1030 | 50 | 100 |
| 底部中央 | 960 | 50 | 50 | 0 |

### 常见误区

```json
// 错误：使用左上角坐标系思维理解 Y 值（这里用绝对像素模式）
"pos_data": { "items": [1820, 980, 95, 90, 0, 0] }
// 这会更接近右上区域，而不是右下角

// 正确：右下角应使用较小的 Y 值
"pos_data": { "items": [1820, 50, 95, 5, 0, 0] }
```

## 锚点

默认情况下，控件锚点在自身中心，也就是 `(0.5, 0.5)`。

这里的锚点值是相对于控件自身宽高的比例值：

- `(0, 0)` 表示左下角
- `(0.5, 0.5)` 表示中心
- `(1, 1)` 表示右上角

需要特别区分两件事：

- `anchor` 是锚点比例值，常规用法是 `0~1`，但源码未做范围限制，因此也可以是负数或大于 `1`
- `set_pos(x, y)` 设置的是当前控件锚点相对于父控件左下角的位置坐标，而不是锚点值本身

因此，如果一个尺寸为 `100x50` 的子控件保持默认中心锚点 `(0.5, 0.5)`，想让它的左下角贴住父控件左下角，应该设置的是位置坐标 `(50, 25)`，不是把锚点改成 `(50, 25)`。

如果真的想把锚点改为左下角，那么应设置为 `(0, 0)`，此时再把控件坐标设为 `(0, 0)`。

```lua
local child = parent:create_child('图片')
child:set_ui_size(100, 50)
-- 这里的 (50, 25) 是位置，不是锚点
child:set_pos(50, 25)
```

如果要在宽度为 `200` 的父控件内让该子控件水平居中、底部对齐：

```lua
local child = parent:create_child('图片')
child:set_ui_size(100, 50)
child:set_pos(100, 25)
```

## 不同分辨率下的适配策略

适配在 cocos 屏幕适配 `cc.RESOLUTIONPOLICY_EXACT_FIT`（以下用 REF 指代）方案的基础上进行。

### 变量定义

- 设计分辨率：`(dw, dh)`
- 实际屏幕分辨率（窗口大小）：`(rw, rh)`
- 横向缩放系数：`sx`
- 纵向缩放系数：`sy`

### 适配规则

- UI 编辑器内的画布尺寸，高度固定为 `1080`，即 `dh = 1080`
- REF 方案下：`sx = rw / dw`，`sy = rh / dh`
- 控件不能变形，必须 `sx = sy`
- 当分辨率改变时，默认缩放控件尺寸与坐标：`sy = rh / 1080`
- 推导：`dw = 1080 / rh * rw`

### 设计分辨率公式

```text
(dw, dh) = (1080 / rh * rw, 1080)
```

游戏内画布大小（ECA 输出值）为 `(1080 / rh * rw, 1080)`，所有子控件的尺寸坐标都基于此。

### 坐标计算流程

1. 设计像素坐标 + 设计分辨率 → 坐标百分比
2. 坐标百分比 + 游戏窗口分辨率 → 实际绝对坐标

例如子控件坐标百分比为 `(xp, yp)`，绝对坐标为 `(1080 / rh * rw * xp, 1080 * yp)`。

## 父子 UI 的坐标关系

子控件的坐标是相对于父控件的**左下角**，而非父控件的锚点位置。控件自身的锚点默认在中心 `(0.5, 0.5)`。

**重要：** `set_pos(x, y)` 设置的是子控件锚点相对于父控件左下角的位置。

```lua
-- 示例：在父控件内放置一个 100x50 的子控件，使其左下角与父控件左下角对齐
local child = parent:create_child('图片')
child:set_ui_size(100, 50)
-- 锚点在中心，所以 x = 宽度/2, y = 高度/2
child:set_pos(50, 25)

-- 示例：让子控件水平居中于宽度为 200 的父控件底部
local child = parent:create_child('图片')
child:set_ui_size(100, 50)
-- x = 父控件宽度/2 = 100（水平居中）
-- y = 子控件高度/2 = 25（底部对齐）
child:set_pos(100, 25)
```

## 核心原则

### 1. 总是优先使用百分比坐标

百分比坐标能够自动适配不同分辨率，避免因画布宽度动态变化导致的布局问题。

### 2. 父边缘吸附优先于手算坐标

如果需求是贴顶、贴底、贴左、贴右、四边撑满，优先用相对父边缘定位，或者编辑器中的适配选项，不要手动维护中心点坐标。

### 3. 适配的源头是边距，不是中心点

一旦开启父边缘适配，`adapter_option` 中记录的边距才是核心数据，`size` 和 `pos_data` 是根据边距反推出来的结果。

---

## 编辑器适配字段

### open_adapter

| 值 | 含义 |
|----|------|
| `true` | 启用适配系统 |
| `false` 或不存在 | 不启用父边缘适配 |

### adapter_option

`adapter_option` 是一个长度为 8 的数组：

```text
adapter_option: [上开关, 下开关, 左开关, 右开关, 上边距, 下边距, 左边距, 右边距]
```

| 索引 | 含义 | 类型 | 说明 |
|------|------|------|------|
| `[0]` | 上适配开关 | `bool` | 基于父节点顶部适配 |
| `[1]` | 下适配开关 | `bool` | 基于父节点底部适配 |
| `[2]` | 左适配开关 | `bool` | 基于父节点左侧适配 |
| `[3]` | 右适配开关 | `bool` | 基于父节点右侧适配 |
| `[4]` | 上边距 | `float` | 控件顶边到父节点顶边的距离 |
| `[5]` | 下边距 | `float` | 控件底边到父节点底边的距离 |
| `[6]` | 左边距 | `float` | 控件左边到父节点左边的距离 |
| `[7]` | 右边距 | `float` | 控件右边到父节点右边的距离 |

### size 和 pos_data 的关系

当适配打开后：

- `adapter_option` 决定边缘约束。
- 引擎根据边距反推出 `size`。
- 引擎再根据尺寸与边距反推出 `pos_data`。

也就是说，边距是约束条件，位置和尺寸是计算结果。

---

## 边距计算公式

假设父节点尺寸为 `(parent_w, parent_h)`，控件中心为 `(cx, cy)`，控件尺寸为 `(w, h)`。

### 由位置和尺寸推导边距

```text
上边距 = parent_h - (cy + h/2)
下边距 = cy - h/2
左边距 = cx - w/2
右边距 = parent_w - (cx + w/2)
```

### 由边距反推尺寸和位置

```text
宽度 w = parent_w - 左边距 - 右边距
高度 h = parent_h - 上边距 - 下边距
中心X cx = 左边距 + w/2
中心Y cy = 下边距 + h/2
```

### 示例 1：四边等距居中

控件中心 `(960, 540)`，尺寸 `100x100`，父节点尺寸 `1920x1080`：

```text
上边距 = 1080 - (540 + 50) = 490
下边距 = 540 - 50 = 490
左边距 = 960 - 50 = 910
右边距 = 1920 - (960 + 50) = 910
```

对应字段：

```json
"adapter_option": [true, true, true, true, 490.0, 490.0, 910.0, 910.0]
```

### 示例 2：非对称边距

若设置上 `400`、下 `500`、左 `600`、右 `700`，父节点为 `1920x1080`：

```text
宽度 = 1920 - 600 - 700 = 620
高度 = 1080 - 400 - 500 = 180
中心X = 600 + 620/2 = 910
中心Y = 500 + 180/2 = 590
```

引擎推导结果：

```json
{
    "adapter_option": [true, true, true, true, 400.0, 500.0, 600.0, 700.0],
    "size": {"__tuple__": true, "items": [620.0, 180.0]},
    "pos_data": {"__tuple__": true, "items": [910.0, 590.0, 47.3958, 54.6296, 0, 0]}
}
```

## 适配行为说明

### 四边全开

```json
"adapter_option": [true, true, true, true, 上边距, 下边距, 左边距, 右边距]
```

父节点尺寸变化时，控件会被拉伸，以保持四条边距离恒定。适合全屏背景、全屏蒙版、撑满容器的布局。

### 仅开左右

```json
"adapter_option": [false, false, true, true, ..., ..., 左边距, 右边距]
```

水平方向保持左右边距不变，宽度会随父节点变化；垂直方向维持原逻辑。

### 仅开上下

```json
"adapter_option": [true, true, false, false, 上边距, 下边距, ..., ...]
```

高度由上下边距共同决定，宽度不变。适合左右尺寸固定、上下拉伸的区域。

### 只开一个方向

```json
"adapter_option": [true, false, false, false, 上边距, ..., ..., ...]
```

控件仅保持距离父节点顶部的间距不变，适合顶部标题、角标、HUD 提示。

### 全关

```json
"adapter_option": [false, false, false, false, 上边距, 下边距, 左边距, 右边距]
```

不进行父边缘适配，控件主要依赖 `pos_data` 中的像素或百分比坐标。边距值可能仍被记录，但运行时不作为约束来源。

## 嵌套适配

父子控件可以各自独立开启适配，且每一层的边距都相对于自己的父节点尺寸计算。

```text
panel（窗口 1920x1080）
  └─ layout_A（相对窗口做适配）
       └─ image_B（相对 layout_A 做适配）
```

如果 `layout_A` 适配后的尺寸变成 `620x180`，那么 `image_B` 的 `adapter_option` 边距就是基于 `620x180` 重新解释，而不是仍然按全屏 `1920x1080` 计算。

这也是为什么嵌套布局里，推荐先把容器层的适配关系设计清楚，再摆放子元素。

## Lua API

本节只讨论 `maps/EntryMap/script/y3` 中可直接调用的接口；`open_adapter`、`adapter_option`、`pos_data` 这类名字不属于 Lua API。

### 坐标与定位相关 API

| 方法 | 参照系 | 说明 |
|------|--------|------|
| `ui:set_pos(x, y)` | 相对父控件 | 设置设计像素坐标，不适合动态宽度下的居中计算 |
| `ui:set_absolute_pos(x, y)` | 相对游戏窗口 | 设置绝对坐标 |
| `GameAPI.set_ui_comp_pos_percent(player.handle, ui.handle, x, y)` | 相对父控件 | 底层 GameAPI，设置 `0-100` 百分比坐标，推荐用于居中布局 |
| `ui:set_relative_parent_pos(direction, offset)` | 相对父控件边缘 | 设置当前控件某条边距离父控件对应边的距离 |

### 相对父控件边缘定位

```lua
---@param direction y3.Const.UIRelativeParentPosType  -- '顶部' | '底部' | '左侧' | '右侧'
---@param offset number
ui:set_relative_parent_pos(direction, offset)
```

| direction | 含义 |
|-----------|------|
| `'顶部'` | 当前控件顶边距离父控件顶边的距离 |
| `'底部'` | 当前控件底边距离父控件底边的距离 |
| `'左侧'` | 当前控件左边距离父控件左边的距离 |
| `'右侧'` | 当前控件右边距离父控件右边的距离 |

全屏铺满示例：

```lua
local panel = root:create_child('空节点')
panel:set_relative_parent_pos('顶部', 0)
panel:set_relative_parent_pos('底部', 0)
panel:set_relative_parent_pos('左侧', 0)
panel:set_relative_parent_pos('右侧', 0)
```

### 百分比坐标定位

```lua
GameAPI.set_ui_comp_pos_percent(player.handle, ui.handle, x, y)
```

- `x`、`y` 范围为 `0-100`
- `50, 50` 表示相对父控件中心

居中示例：

```lua
local btn = root:create_child('按钮')
GameAPI.set_ui_comp_pos_percent(player.handle, btn.handle, 50, 50)
```

### 画布尺寸获取

```lua
local width = y3.ui.get_screen_width()
local height = y3.ui.get_screen_height()
```

说明：

- Lua 层接口分别对应 `y3.ui.get_screen_width()` / `y3.ui.get_screen_height()`
- 文档中的“高度固定为 `1080`、宽度随窗口宽高比变化”属于当前项目经验结论，不是单靠 Lua 签名就能证明的接口契约
- 不建议拿 `get_screen_width()` 去计算 `set_pos` 的居中位置
- 想做居中时，应优先使用百分比坐标

### 文本对齐方式

```lua
---@param h y3.Const.UIHAlignmentType  -- '左' | '中' | '右'
---@param v y3.Const.UIVAlignmentType  -- '上' | '中' | '下'
ui:set_text_alignment(h, v)
```

```lua
local title = root:create_child('文本')
title:set_ui_size(400, 60)
title:set_text_alignment('中', '中')
```


## 常见问题与解决方案

### 问题 1：控件无法水平居中

错误做法：

```lua
local screen_width = y3.ui.get_screen_width()
ui:set_pos(screen_width / 2, 540)
```

问题在于 `set_pos` 使用的是设计坐标，而 `screen_width` 是当前运行时画布宽度，语义并不一致。

正确做法：

```lua
GameAPI.set_ui_comp_pos_percent(player.handle, ui.handle, 50, 50)
```

### 问题 2：全屏背景无法铺满

正确做法是四边吸附：

```lua
local bg = root:create_child('图片')
bg:set_relative_parent_pos('顶部', 0)
bg:set_relative_parent_pos('底部', 0)
bg:set_relative_parent_pos('左侧', 0)
bg:set_relative_parent_pos('右侧', 0)
bg:set_image(图片资源ID)
```

### 问题 3：文本显示不全

文本控件显示不全通常不是坐标问题，而是控件尺寸和对齐方式不合适。

```lua
local title = root:create_child('文本')
title:set_text('碰碰车大作战')
title:set_font_size(48)
title:set_ui_size(400, 60)
title:set_text_alignment('中', '中')
```

### 问题 4：图片控件显示默认图

仅设置颜色而不设置图片资源时，可能会显示默认图。

```lua
local bg = root:create_child('图片')
bg:set_image(图片资源ID)
bg:set_image_color(180, 180, 180, 255)
```

### 问题 5：按钮文字不清晰

优先检查背景图片叠加色和文字色的对比度。

```lua
local btn = root:create_child('按钮')
btn:set_image_color(50, 150, 50, 255)
btn:set_text_color(255, 255, 255, 255)
```

## 推荐布局模式

以下模式分成两层理解：

- 如果你在说**编辑器资源数据**，优先看 `adapter_option` 这类字段。
- 如果你在说**Lua 运行时代码**，优先看 `set_pos`、`set_absolute_pos`、`GameAPI.set_ui_comp_pos_percent(...)`、`set_relative_parent_pos` 这些接口。

### 全屏容器 + 百分比居中

适用于主菜单、结算弹窗、居中按钮组。

```lua
local root_panel = canvas:create_child('空节点')
root_panel:set_relative_parent_pos('顶部', 0)
root_panel:set_relative_parent_pos('底部', 0)
root_panel:set_relative_parent_pos('左侧', 0)
root_panel:set_relative_parent_pos('右侧', 0)

local bg = root_panel:create_child('图片')
bg:set_relative_parent_pos('顶部', 0)
bg:set_relative_parent_pos('底部', 0)
bg:set_relative_parent_pos('左侧', 0)
bg:set_relative_parent_pos('右侧', 0)

local btn = root_panel:create_child('按钮')
```

### 编辑器内常用开关组合

| 场景 | 上 | 下 | 左 | 右 | 效果 |
|------|----|----|----|----|------|
| 全屏铺满 | 是 | 是 | 是 | 是 | 四边距设为 `0`，自动撑满父节点 |
| 顶部固定栏 | 是 | 否 | 是 | 是 | 贴顶，左右撑满，高度固定 |
| 底部固定栏 | 否 | 是 | 是 | 是 | 贴底，左右撑满，高度固定 |
| 左侧面板 | 是 | 是 | 是 | 否 | 贴左，上下撑满，宽度固定 |
| 右侧面板 | 是 | 是 | 否 | 是 | 贴右，上下撑满，宽度固定 |
| 左上角固定 | 是 | 否 | 是 | 否 | 保持左上角边距 |
| 右上角固定 | 是 | 否 | 否 | 是 | 保持右上角边距 |
| 居中不适配 | 否 | 否 | 否 | 否 | 依赖 `pos_data` 或百分比定位 |

## 实践建议

1. 全屏背景、遮罩、容器底板优先四边吸附，避免手动计算尺寸。
2. 居中元素优先用百分比坐标，不要用 `get_screen_width()` 配合 `set_pos` 手算。
3. 角落 HUD 优先只锁定对应边缘，例如左上角常用上 + 左。
4. 多层嵌套界面先把容器适配关系理顺，再布置内部子控件。
5. 如果一个控件既要保持边距又要随父节点拉伸，编辑器资源层优先使用 `adapter_option`；Lua 运行时可以结合 `set_relative_parent_pos` 实现对应边缘吸附，不要只改 `pos_data`。
