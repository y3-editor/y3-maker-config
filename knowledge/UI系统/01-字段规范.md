# UI 字段规范

本文档记录 Y3 编辑器 UI JSON 中需要严格遵守的字段规范。

---

## Y3 坐标系统（必读！）

### 坐标系说明

Y3 使用**左下角原点**坐标系：
- **Y=0%, Y_ABS=0** → 屏幕**底部**
- **Y=100%, Y_ABS=1080** → 屏幕**顶部**
- **X=0%** → 屏幕**左侧**
- **X=100%** → 屏幕**右侧**

### 常用布局参考值（从 [HUD]Main.json 提取）

| 区域 | 组件示例 | pos_data | 说明 |
|------|----------|----------|------|
| **底部 HUD** | MainConsole | `[960,85] 50%,8%` | Y%=8% 表示距底部 8% |
| **顶部栏** | [TOP]HeadInfo | `[960,1046] 50%,97%` | Y%=97% 表示距底部 97% |
| **左上角** | [LEFT]Map | `[130,155] 7%,14%` | 小地图区域 |
| **右侧** | [RIGHT]Statistic | `[1640,833] 85%,77%` | 统计面板 |

### 禁止臆造坐标值！

**错误做法**：
```python
# 错误：凭感觉设置坐标
"pos_data": {"__tuple__": True, "items": [960.0, 120.0, 50.0, 11.1, 1, 1]}
```

**正确做法**：
```python
# 正确：参考 [HUD]Main.json 中的 MainConsole
# MainConsole: pos=[960,85] 50%,8%
"pos_data": {"__tuple__": True, "items": [960.0, 85.0, 50.0, 8.0, 1, 1]}
```

### 如何获取正确的坐标值

1. 打开 `maps/EntryMap/ui/[HUD]Main.json`
2. 找到功能相似的组件
3. 复制其 `pos_data` 值
4. 根据需要微调

## 致命字段（非法值会导致编辑器崩溃）

这些字段如果使用非法值，会导致 **KeyError** 或 **IndexError**，使地图无法打开。

### 1. pos_data（定位数据）

```python
# 格式
pos_data = [X_ABS, Y_ABS, X_PCT, Y_PCT, X_MODE, Y_MODE]

# 索引定义 (UIPosIndex)
X_ABS = 0    # X 绝对位置（像素）
Y_ABS = 1    # Y 绝对位置（像素）
X_PCT = 2    # X 百分比位置（%）
Y_PCT = 3    # Y 百分比位置（%）
X_MODE = 4   # X 定位模式
Y_MODE = 5   # Y 定位模式

# 定位模式 (UIPosMode)
ABS = 0  # 像素定位
PCT = 1  # 百分比定位

# ui_const.py 中的默认值
UI_POS_DATA_ABS_DEF = (None, None, 50, 50, 0, 0)
UI_POS_DATA_PCT_DEF = (0, 0, 50, 50, 1, 1)
```

| 字段 | 有效值 | 说明 |
|------|--------|------|
| X_MODE | `0` 或 `1` | 0=像素, 1=百分比 |
| Y_MODE | `0` 或 `1` | 0=像素, 1=百分比 |

**致命**: 使用 `2` 或其他值作为 MODE 会触发 `KeyError: 2`

---

### 2. alignment（对齐方式）

```python
# 格式
alignment = [H_ALIGN, V_ALIGN]

# 水平对齐
H_LEFT = 1
H_CENTER = 2
H_RIGHT = 4

# 垂直对齐
V_TOP = 0
V_CENTER = 8
V_BOTTOM = 16
```

| 字段 | 有效值 | 说明 |
|------|--------|------|
| H_ALIGN | `1`, `2`, `4` | 1=左, 2=中, 4=右 |
| V_ALIGN | `0`, `8`, `16` | 0=顶, 8=中, 16=底 |

补充说明：

- `alignment` 表示内容或节点的对齐语义，不等于控件锚点
- 以编辑器源码与运行时常量为准，垂直对齐值应按 `顶=0 / 中=8 / 底=16` 理解

---

### 3. open_adapter / adapter_option（适配选项）

```python
# 是否开启适配系统
open_adapter = true | false

# EditorUICompMeta.py 中的默认值
open_adapter = False
adapter_option = [False, False, False, False, 0, 0, 0, 0]
```

- `true`: 启用适配系统，`adapter_option` 生效
- `false` 或不存在: 不启用适配，控件主要按 `pos_data` 定位

```python
# 格式（8元素数组）
adapter_option = [TOP, BOTTOM, LEFT, RIGHT, TOP_MARGIN, BOTTOM_MARGIN, LEFT_MARGIN, RIGHT_MARGIN]

# 索引 0-3: 布尔值，控制是否启用对应边的适配
# 索引 4-7: 浮点数，表示当前控件对应方向边框到父节点对应方向边框的距离
```

关键规则：

- 这里的“边距”指当前控件的上/下/左/右边框，分别到父节点同方向边框的距离
- 开启适配后，`size` 和 `pos_data` 会由引擎根据边距自动回算
- 也就是说，边距是源头，尺寸和位置是结果
- 不需要适配时，可以不设置 `open_adapter`，直接使用 `pos_data`
- 详细边距公式、适配模式和示例，以 `skills/y3-ui-generator/references/adapter-rules.md` 为准

---

### 4. cap_insets / *_cap_insets（九宫格切片）

```python
# 格式（4元素数组）
cap_insets = [LEFT, TOP, RIGHT, BOTTOM]
```

源码中当前可见的常见切片字段：

- `Image.cap_insets`，默认值 `(-1, -1, 0, 0)`
- `Button.normal_cap_insets / suspend_cap_insets / press_cap_insets / disabled_cap_insets`，默认值 `(24, 8, 24, 8)`
- `Progress.background_cap_insets` 与 `cap_insets`

九宫格切片不是只有一个统一字段名；不同组件会使用各自的 `cap_insets` 字段。

### 5. size（尺寸）

```python
# 格式（2元素数组）
size = [WIDTH, HEIGHT]
```
