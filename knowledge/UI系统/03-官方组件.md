# Y3 官方 UI 组件指南

## 核心原则

不要重复手写引擎已经封装好的 UI 组件。技能栏、Buff 栏、物品槽等通用结构，优先使用官方组件，而不是手拼一套图片加逻辑。

对于官方组件，如果确实需要调整其子控件，建议最多只修改控件引用的图片资源；不要修改影响适配的参数，例如坐标、`pos_data`、`open_adapter`、`adapter_option` 等。

## 技能按钮组件（type: 17）

### 组件结构

```json
{
    "name": "skill_btn_1",
    "type": 17,
    "skill_type": 3,
    "slot_id": 1,
    "skill_icon_img": "...",
    "skill_cd_progress": "...",
    "skill_cd_label": "...",
    "skill_charge_progress": "...",
    "skill_consume_mask_img": "...",
    "skill_disable_img": "...",
    "skill_frame_img": "...",
    "skill_level_label": "...",
    "skill_shortcut_label": "...",
    "skill_stack_label": "...",
    "skill_upgrade_btn": "..."
}
```

### skill_type 枚举

| 值 | 含义 | 对应 AbilityType |
|----|------|------------------|
| 0 | 隐藏技能 | HIDE |
| 1 | 普通技能 | NORMAL |
| 2 | 命令技能 | COMMON |
| 3 | 英雄技能 | HERO |

### 使用方式

1. 在编辑器中添加：组件库 -> 封装组件 -> 技能按钮。
2. 配置 `skill_type` 和 `slot_id`。
3. 运行时通过选中单位或手动绑定技能来刷新内容。

### 代码示例

```lua
player:select_unit(hero)

local skill_btn = y3.ui.get_ui(player, "skill_btn_1")
skill_btn:bind_ability(ability)
skill_btn:set_skill_btn_smart_cast_key(y3.const.KeyboardKey['Q'])
```

### 关键字段

```json
{
    "name": "skill_btn_1",
    "type": 17,
    "skill_type": 3,
    "slot_id": 1,
    "swallow_touches": true,
    "skill_icon_img": "xxx-uid",
    "skill_cd_progress": "xxx-uid",
    "skill_cd_label": "xxx-uid",
    "skill_charge_progress": "xxx-uid",
    "skill_consume_mask_img": "xxx-uid",
    "skill_disable_img": "xxx-uid",
    "skill_frame_img": "xxx-uid",
    "skill_level_label": "xxx-uid",
    "skill_shortcut_label": "xxx-uid",
    "skill_stack_label": "xxx-uid",
    "skill_upgrade_btn": "xxx-uid"
}
```

### 必需子节点

| 子节点名称 | type | 用途 |
|-----------|------|------|
| `skill_icon_img` | 4 | 技能图标 |
| `skill_frame_img` | 4 | 边框 |
| `skill_cd_progress` | 5 | 冷却遮罩 |
| `skill_cd_label` | 3 | 冷却倒计时文字 |
| `skill_shortcut_label` | 3 | 快捷键显示 |
| `skill_level_label` | 3 | 技能等级 |
| `skill_stack_label` | 3 | 充能层数 |
| `skill_consume_mask_img` | 4 | 法力不足遮罩 |
| `skill_disable_img` | 4 | 禁用状态图 |
| `skill_upgrade_btn` | 1 | 升级按钮 |
| `skill_charge_progress` | 42 | 充能进度 |

### 生成规则

默认优先使用脚本自动生成；仅在模板缺失或结构异常时，回退到编辑器手动添加。

1. 优先用 `html_to_y3_ui.py` 配合官方组件模板生成。
2. 如生成结果异常，再在编辑器中手动添加同类型组件并导出对照。
3. Lua 层通常只需 `player:select_unit(hero)` 或 `ui:bind_ability(ability)`。

### 错误做法

```lua
local icon = skill_btn:get_child('icon')
icon:set_image(ability:get_icon())

y3.timer.loop(0.1, function()
    local cd = ability:get_cd()
    progress:set_value(cd)
end)
```

这些工作本来就由官方组件负责，不必重复实现。

## 物品槽组件（type: 20）

### 英雄物品栏系统

英雄单位有两类物品槽位：

| 类型 | SlotType | 说明 |
|------|----------|------|
| 物品栏 | `BAR` (1) | 装备位 |
| 背包栏 | `PKG` (0) | 仓库位 |

### 使用方式

```lua
local item_slot = y3.ui.get_ui(player, "item_slot_1")

item_slot:set_ui_unit_slot(hero, y3.const.SlotType.BAR, 0)
item_slot:set_ui_unit_slot(hero, y3.const.SlotType.PKG, 0)
```

### SlotType 枚举

```lua
y3.const.SlotType = {
    NOT_IN_BAG = -1,
    PKG = 0,
    BAR = 1,
}
```

### API 参数说明

`set_ui_unit_slot(unit, field, index)`

| 参数 | 类型 | 说明 |
|------|------|------|
| `unit` | Unit | 要绑定的单位 |
| `field` | SlotType | 槽位类型，`BAR` 或 `PKG` |
| `index` | integer | 格子索引，从 `0` 开始 |

### 引用字段配置

物品槽组件必须正确配置以下引用字段，否则会显示异常：

| 引用字段 | 必须指向 | 说明 |
|----------|----------|------|
| `equip_bg_img` | `bg` 子节点 UID | 背景图 |
| `equip_icon_img` | `icon` 子节点 UID | 物品图标 |
| `equip_cd_progress` | `cd` 子节点 UID | 冷却进度条 |
| `equip_stack_label` | `stack` 子节点 UID | 堆叠数量文本 |
| `equip_disabled_img` | `disabled_mask` 子节点 UID | 禁用遮罩 |
| `hover_child` | `hover_resp` 子节点 UID | 悬停响应区域 |

错误示例：

```json
"equip_bg_img": "none_ref",
"equip_icon_img": "none_ref"
```

正确示例：

```json
"equip_bg_img": "37f90bf5-05f3-431d-89fb-3a1e3cc57f2d",
"equip_icon_img": "b2e86afe-a22a-4055-b715-d6b5d9ac6914"
```

### 模板字段精简规范

官方组件对 JSON 字段较敏感，多余字段可能导致初始化异常。

#### 不建议在官方组件根节点出现的字段

| 字段名 | 说明 |
|--------|------|
| `adapter_option` | 普通控件适配字段 |
| `open_adapter` | 普通控件适配开关 |
| `slot_type` | 槽位类型应通过 Lua 动态指定 |
| `use_operation` | 非必需字段 |

#### 物品槽最小必需字段

```json
{
    "name": "item_slot_1",
    "type": 20,
    "uid": "<新生成的UUID>",
    "layer_name": "<面板名称>",
    "pos_data": { "__tuple__": true, "items": [0, 0, 0, 0, 1, 0] },
    "size": { "__tuple__": true, "items": [64.0, 64.0] },
    "event_list": [],
    "children": [],
    "equip_bg_img": "<bg子节点UID>",
    "equip_icon_img": "<icon子节点UID>",
    "equip_cd_progress": "<cd子节点UID>",
    "equip_stack_label": "<stack子节点UID>",
    "equip_charge_label": "<stack子节点UID>",
    "equip_disabled_img": "<disabled_mask子节点UID>",
    "hover_child": "<hover_resp子节点UID>",
    "mask_image": 999,
    "short_cut": 0
}
```

### 排查方法

1. 在编辑器中手动添加同类型组件并导出。
2. 对比异常组件和正常组件的字段差异。
3. 检查所有 `xxx_img`、`xxx_label`、`hover_child` 是否确实指向 `children` 中存在的 UID。

### 示例：绑定 6 格物品栏与 6 格背包栏

```lua
for i = 1, 6 do
    local slot = y3.ui.get_ui(player, 'hud.item_bar.slot_' .. i)
    slot:set_ui_unit_slot(hero, y3.const.SlotType.BAR, i - 1)
end

for i = 1, 6 do
    local slot = y3.ui.get_ui(player, 'hud.bag.slot_' .. i)
    slot:set_ui_unit_slot(hero, y3.const.SlotType.PKG, i - 1)
end
```
