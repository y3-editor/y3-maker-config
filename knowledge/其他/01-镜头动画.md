# 镜头动画系统 (Camera Animation System)

## 概述

镜头动画系统用于制作镜头过场、拉远拉近、转向和焦点移动等效果。

**文件位置**：`maps\EntryMap\cameraanim.json`

## 编辑器中的基础规则

- 一个动画对象按 `anim_id` 存在于顶层字典中。
- 新建动画时，编辑器会自动创建一个根轨道 `Camera`，并自动挂上 7 条默认子轨道。
- 每条轨道初始化时都会生成 `frame 0` 的首帧；首帧不能删除。
- 编辑器只能在 `frame > 0` 的位置新增关键帧。
- 时间轴预览按 **30fps** 播放。

### 顶层字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `anim_id` | int | 动画 ID |
| `anim_name` | string / null | 动画名称字符串，源码未限定为语言包 ID；未设置时也可以为 `null` |
| `enabled` | bool | 是否启用 |
| `return_first_frame` | bool | 预览播放结束后是否回到首帧 |
| `use_game_last_frame` | bool | 播放前是否先记录当前游戏镜头，并把该状态当作首帧基准 |
| `track_data` | dict | 轨道数据 |

### `track_data` 的实际层级

`track_data` 顶层通常只保存一个根轨道 `Camera`。`Focus`、`Distance`、`Pitch`、`Roll`、`Yaw`、`Fov`、`FarClipping` 都嵌套在根轨道的 `sub_tracks` 中。上一版若把 `sub_tracks` 写成空对象，只能算“结构草图”，不算源码对应的完整序列化结果。

## 轨道系统 (Track)

```json
{
    "track_id": 1,
    "track_name": "Camera",
    "track_desc": "code_camline",
    "track_type": 0,
    "is_loop": false,
    "frame_data": {},
    "sub_tracks": {}
}
```

### 轨道字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `track_id` | int | 轨道 ID |
| `track_name` | string | 轨道内部名称 |
| `track_desc` | string | 编辑器显示文案对应的语言 key |
| `track_type` | int | 轨道类型 |
| `is_loop` | bool | 循环标记；当前已查看的镜头时间轴源码中未见实际播放逻辑使用它 |
| `frame_data` | dict | 当前轨道的关键帧数据 |
| `sub_tracks` | dict | 子轨道 |

### 轨道类型 (`track_type`)

| track_type | track_name | track_desc | 说明 |
|------------|------------|------------|------|
| 0 | Camera | code_camline | 根轨道 / 容器轨道 |
| 1 | Focus | code_attr_camera_focus | 焦点位置 |
| 2 | Distance | code_attr_camera_dist | 镜头距离 |
| 3 | Pitch | code_attr_camera_pitch | 俯仰角 |
| 4 | Roll | code_attr_camera_roll | 翻滚角 |
| 5 | Yaw | code_attr_camera_yaw | 偏航角 |
| 6 | Fov | code_attr_camera_fov | 视野角 |
| 7 | FarClipping | code_attr_camera_far | 远裁剪面 |

### 轨道层级结构

```text
Camera (track_type=0)
├── Focus (track_type=1)
├── Distance (track_type=2)
├── Pitch (track_type=3)
├── Roll (track_type=4)
├── Yaw (track_type=5)
├── Fov (track_type=6)
└── FarClipping (track_type=7)
```

## 关键帧数据 (`frame_data`)

```json
{
    "26": {
        "frame_type": 1,
        "frame_value": 164.2,
        "value_type": 1,
        "curve_type": 0,
        "curve_id": null
    }
}
```

### 关键帧字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `frame_type` | int | 帧类型 |
| `frame_value` | number / tuple / null | 帧值；容器轨道通常为空 |
| `value_type` | int | 值类型 |
| `curve_type` | int | 插值曲线类型 |
| `curve_id` | int / null | 自定义曲线 ID；仅 `curve_type = -1` 时有效 |

### 帧类型 (`frame_type`)

| frame_type | 名称 | 说明 |
|------------|------|------|
| -1 | First Frame | 首帧，固定出现在 `frame 0` |
| 0 | Default | 占位帧，主要用于父轨道标记该帧存在子轨道数据 |
| 1 | Absolute Frame | 绝对帧，`frame_value` 直接表示目标值 |
| 2 | Relative Frame | 相对帧，`frame_value` 表示“相对首帧的偏移量” |

补充说明：

- 编辑器 UI 里可主动创建的只有 **绝对帧** 和 **相对帧**。
- 首帧由轨道初始化时自动生成。
- 父轨道上的 `Default` 帧通常不参与插值，只用于时间轴显示和层级占位。

### 值类型 (`value_type`)

| value_type | 名称 | 说明 |
|------------|------|------|
| -1 | Default | 容器轨道默认值类型 |
| 0 | Int | 源码中定义了该类型，但当前镜头轨道未使用 |
| 1 | Float | 数值型属性：Distance、Pitch、Roll、Yaw、Fov、FarClipping |
| 2 | Vec3 | 三维向量：Focus |

### `frame_value` 的实际格式

| 轨道 | 存储格式 | 示例 |
|------|----------|------|
| Focus | tuple(x, y, z) | `{"__tuple__": true, "items": [0.0, 0.0, 0.0]}` |
| Distance | float | `164.2` |
| Pitch / Roll / Yaw | float | 以弧度存储 |
| Fov | float | 直接存储当前 FOV 数值 |
| FarClipping | float | 远裁剪距离 |
| Camera | null | 容器轨道无实际数值 |

### 曲线类型 (`curve_type`)

镜头时间轴编辑器当前只暴露 4 种曲线类型：

| curve_type | 名称 | 说明 |
|------------|------|------|
| -1 | Custom Easing | 自定义贝塞尔曲线，需要 `curve_id` |
| 0 | Linear | 线性插值 |
| 1 | EaseIn | 缓入 |
| 2 | EaseOut | 缓出 |

虽然底层 `TweenType` 还定义了更多缓动类型，但当前镜头动画属性面板只提供上表这 4 种。

## 数值单位与属性面板显示

源码里“存储值”和“属性面板显示值”并不完全一致：

| 轨道 | 文件中存储 | 属性面板显示 | 备注 |
|------|------------|--------------|------|
| Focus | `(x, y, z)` | `(x, z, y)`，且每个分量乘以 100 | 面板中坐标顺序和内部顺序不同 |
| Distance | float | 存储值 × 100 | 面板滑条范围 `0 ~ 48000` |
| Pitch | 弧度 | 角度 | 面板范围 `-89.99 ~ 89.99` |
| Roll | 弧度 | 角度 | 面板范围 `0 ~ 360` |
| Yaw | 弧度 | 角度 | 面板范围 `0 ~ 720` |
| Fov | float | 直接显示 | 面板范围 `20 ~ 120` |
| FarClipping | float | 存储值 × 100 | 面板范围 `500 ~ 4800000` |

`Focus` 的 X/Z 范围还会随地图尺寸变化：

- X 范围：`[-地图宽度 / 2, 地图宽度 / 2] * 100`
- Z 范围：`[-地图高度 / 2, 地图高度 / 2] * 100`
- Y 范围固定：`-48000 ~ 48000`

## 编辑器操作补充

### 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl + A` | 在当前帧添加绝对帧 |
| `Ctrl + R` | 在当前帧添加相对帧 |
| `Delete` | 删除当前帧的关键帧（首帧除外） |
| `Ctrl + Z` | 撤销 |
| `← / →` | 切换当前帧 |
| `↑ / ↓` | 切换当前轨道 |

### 选中根轨道时的属性行为

如果当前选中的是根轨道 `Camera`，属性面板不会编辑根轨道本身，而是把当前帧下存在关键帧的所有子轨道属性一起展示出来。

## 相对帧与插值规则

- 绝对帧：`frame_value` 就是该帧的目标值。
- 相对帧：实际值 = `首帧值 + 当前帧偏移值`。
- 插值总是在“前一个已有关键帧”和“后一个已有关键帧”之间计算。
- 若起始关键帧使用自定义曲线，则会按 `curve_id` 对应的贝塞尔参数插值。

## 地图尺寸变化时的自动修正

当地图宽高变化时，编辑器会自动调整 `Focus` 轨道：

- 首帧焦点会按地图边界变化量整体平移。
- 如果平移后超出新地图范围，首帧会被重置到 `(0, 原Y, 0)`。
- `Absolute Frame` 会保留“相对首帧的偏移”，再基于新首帧重新计算。
- `Relative Frame` 不会被重写。

## y3 LuaLib 脚本接口

按当前项目的 `maps\EntryMap\script\y3` 核对，镜头动画相关的 Lua 对外接口为：

```lua
-- 播放镜头动画
y3.camera.play_camera_timeline(player, camera_timeline_id)

-- 停止镜头动画
y3.camera.stop_camera_timeline(player)

-- 判断玩家镜头是否正在播放 timeline
y3.camera.is_camera_playing_timeline(player)
```

### 接口说明

| 文档中的调用意图 | y3 LuaLib 实际接口 | 参数名 |
|------------------|--------------------|--------|
| 播放镜头动画 | `y3.camera.play_camera_timeline` | `player`, `camera_timeline_id` |
| 停止镜头动画 | `y3.camera.stop_camera_timeline` | `player` |
| 判断是否正在播放 | `y3.camera.is_camera_playing_timeline` | `player` |
