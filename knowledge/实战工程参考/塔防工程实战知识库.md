## 1. 工程结构

```
maps/EntryMap/script/
├── main.lua                    ← 引擎入口
├── 可重载的代码.lua             ← 热重载入口（include 6 模块 + 三层触发）
├── td_config.lua               ← 全局数值配置（塔/怪物/波次/升级/路径）
├── td_state.lua                ← 游戏状态机（金币/生命/波次/胜负判定）
├── td_build.lua                ← 建造/升级/出售逻辑
├── td_game.lua                 ← 主游戏循环（波次刷怪/路径移动/事件监听/定时器管理）
├── td_extras.lua               ← 特殊效果（减速/AOE/冰霜机制）
└── td_ui.lua                   ← UI 交互（状态栏/塔选择/升级面板/结算/区域点击）
```

> **架构原则**：单一职责——config 只写配置、state 只管状态、build 只管建造、game 只管主循环、extras 只管特殊效果、ui 只管界面。

## 3. 物编配置关键陷阱

### 3.1 攻击类型

```
❌ 建筑模板(jz.json) → common_atk_type: 0 (近战)
✅ 防御塔 required → common_atk_type: 1 (远程)
```

修复命令：
```bash
python3 y3_obj_edit.py -map maps/EntryMap -type unit -edit {id} -common_atk_type 1
# 500001, 500002, 500003, 500004 全部执行
```

### 3.2 攻击特效配置

每个塔的普攻技能需配置 `simple_common_atk.attack_trajectory.effect`（弹道特效）和 `hit_effect.effect`（受击特效）：

| 塔 | 弹道特效 | 受击特效 |
|----|---------|---------|
| 箭塔 | 104656 绿色箭 | 104657 绿色毒 |
| 魔法塔 | 102892 紫色暗影球 | 104982 紫色暗影 |
| 炮塔 | 104673 橙色火球拖尾 | 105110 橙色火焰 |
| 冰塔 | 104378 蓝色水球 | 102295 蓝色冰霜 |

### 3.3 `ori_speed` ×100 存储

物编 `ori_speed` 引擎内部 ×100 存储。物编写 100 → `get_attr('ori_speed','基础')` 返回 10000。需在 spawn 时 `set_attr('移动速度', cm_per_s, '基础')` 归一化。

## 4. 核心模块职责与关键代码

### 4.1 入口与热重载（`可重载的代码.lua`）

```lua
-- 三层触发：正常启动 / 编辑器直跑 / 热重载
y3.game:event('游戏-初始化', init_td)
y3.ltimer.wait(0.1, function() init_td() end)
y3.reload.onAfterReload(function(_, has_reloaded)
    if has_reloaded then init_td() end
end)
```

### 4.2 init() 幂等性（必须 4 项清理）

```lua
function M.init(player)
    -- 1. 清理旧 timer
    for _, t in ipairs(M.timers) do if t then t:remove() end end
    M.timers = {}
    -- 2. 清理旧单位
    if M.base_unit and M.base_unit:is_exist() then M.base_unit:remove() end
    for _, m in ipairs(M.alive_monsters) do if m:is_exist() then m:remove() end end
    M.alive_monsters = {}
    -- 3. 重置状态
    State.reset()
    -- 4. 重新获取路径（重载后旧引用失效）
    ROAD = get_road()
end
```

### 4.3 事件防重复注册

```lua
M._events_registered = false
function M.register_events(player)
    if M._events_registered then return end
    y3.game:event('技能-建造完成', callback)
    M._events_registered = true
end
```

### 4.4 路径系统

```
获取: y3.road.get_road_by_res_id(10000)     ← 编辑器静态摆放
移动: unit:move_along_road(road, '单向', false, true, false)
检测: ltimer.loop(0.5, dist-check)           ← '单位-移动结束' 事件不可靠
兜底: td_config.PATH_POINTS 写死坐标
```

### 4.5 数值集中管理（`td_config.lua`）

所有可调数值在一个文件中：
- 塔属性（cost/damage/range/speed）
- 波次配置（WAVE_INTERVAL / SPAWN_INTERVAL）
- 初始资源（金币/生命）
- 升级成本 / 比例
- 路径坐标

> 改数值 → `.rd` 热重载 → 即时生效，无需重启。

### 4.6 set_attr / get_attr 正确用法

```lua
-- ✅ 合法
tower:set_attr('物理攻击', 50, '基础')
enemy:set_attr('移动速度', 200, '基础')
local atk = tower:get_attr('物理攻击', '基础')

-- ❌ 非法（不报错但不生效）
tower:set_attr('atk_base', 50, '基础')
tower:set_attr('ATTACK_PHY', 50, '基础')

-- 必须用 set_attr (覆盖物编模板值) 非 add_attr (叠加)
```

属性类型：`'基础'` / `'基础加成'` / `'增益'` / `'增益加成'` / `'最终加成'`

### 4.7 建造技能方案

```
ability_cast_type = 1  (普通技能，非 4——Y3 不支持真正的建造技能指示器)
sight_type = 0          (无指示器)
is_immediate = true     (立即释放，点击按钮 → 进入建造模式)
build_list              (产出单位 ID)
common_ability_list     挂在基地单位上
```

Lua 监听：`'技能-建造完成'` → `data.build_unit`（不是 `data.unit`）

### 4.8 禁选 + 常驻选中

```lua
player:select_unit(base_unit)
player:set_all_operation_key(1, false)  -- 禁用"选择单位"
player:set_all_operation_key(2, false)  -- 禁用"框选"
```

## 5. Bug 修复记录（7 条，全部已修复）

| # | Bug | 根因 | 修复 |
|---|-----|------|------|
| 1 | 重新开始时旧 timer 继续跑 | `ltimer.loop` 未追踪 | `M.timers` 追踪，init 遍历 remove |
| 2 | 升级后攻击力不变 | `'atk_base'` 不是合法 UnitAttr 键 | 改用 `'物理攻击'` 中文键 |
| 3 | 怪物数量过多 | WAVE_INTERVAL=5 / SPAWN_INTERVAL=0.8 太密 | 改为 12s / 1.2s，金币 100→200 |
| 4 | 怪物移速起飞 | `'ORI_SPEED'`=施法速度 | 改用 `'移动速度'` |
| 5 | 冰塔减速无限叠加 | 重复攻击在当前减速值上再减半 | 首次记录 origin_speed，重复只重置 timer |
| 6 | 升级按钮双击升两级 | init 被调两次 → player:event 叠加两个回调 | `_xxx_registered` 防重复 |
| 7 | 防御塔攻击类型全是近战 | 建筑模板 common_atk_type=0 | 改为 1(远程) |

## 6. 数值迭代工作流

```
修改 td_config.lua 任意数值
        ↓
send .rd（或 y3.reload.reload()）
        ↓
游戏自动重新初始化，加载新数值
        ↓
立即验证效果
```

**常见调整项及位置**：
- 塔伤害/费用/射程 → `td_config.lua` §TOWER_DATA
- 波次间隔/怪物数量 → `td_config.lua` §WAVE_INTERVAL / SPAWN_INTERVAL
- 升级提升比例 → `td_config.lua` §UPGRADE_DAMAGE_RATIO
- 初始资源 → `td_config.lua` §START_GOLD / START_LIVES
- 怪物移速 → `td_config.lua` → `td_game.lua:124` spawn 时 `set_attr('移动速度',...)`

## 8. 补充机制与 API 用法

### 8.1 AOE 防递归

炮塔 AOE 通过 `'单位-造成伤害后'` 事件触发 → AOE 对其他怪物造成伤害 → 再次触发事件 → 无限递归。解法：

```lua
M._dealing_aoe = false

function M.deal_aoe_damage(attacker, target)
    if M._dealing_aoe then return end  -- 关键！防递归
    M._dealing_aoe = true
    -- ... 遍历 alive_monsters 造成伤害 ...
    M._dealing_aoe = false
end
```

### 8.2 `unit:damage()` 参数结构

```lua
attacker:damage({
    target = enemy,         -- Unit: 受伤目标
    damage = 80,            -- number: 伤害值
    damage_type = 1,        -- 1=物理 2=魔法（参见 knowledge/核心系统/07-伤害类型.md）
})
```

> 来源：`td_extras.lua:60`，已验证可用。

### 8.3 `add_fast_event` 用法（UI 按钮绑定）

```lua
local btn = y3.ui.get_ui(player, "PanelName.block.button_name")
btn:add_fast_event('左键-按下', function(trg)
    -- 按钮被点击
end)
```

| 对比 | `add_fast_event` | `player:event(...)` |
|------|-----------------|---------------------|
| 绑定对象 | 具体 UI 控件 | 全局 / 玩家级 |
| 触发粒度 | 仅该控件点击 | 需在回调内判断来源 |
| 推荐场景 | 按钮/面板交互 | 全局键盘/鼠标事件 |

### 8.4 `鼠标-按下单位` 事件格式

```lua
y3.game:event('鼠标-按下单位', y3.const.MouseKey['LEFT'], function(trg, data)
    local unit = data.unit   -- 被点击的单位
    -- ...
end)
```

> 第二参数为鼠标键常量，`'LEFT'` / `'RIGHT'` / `'MIDDLE'`。

### 8.6 重新开始清理链

ResultPanel 的 restart 按钮触发时的完整清理流程：

```lua
-- 1. 清理旧怪物
for _, m in ipairs(Game.alive_monsters) do
    if m and m:is_exist() then m:remove() end
end
Game.alive_monsters = {}

-- 2. 清理所有防御塔
for tower, _ in pairs(Build.tower_levels) do
    if tower and tower:is_exist() then tower:remove() end
end
Build.tower_levels = {}

-- 3. 清理基地
if Game.base_unit and Game.base_unit:is_exist() then
    Game.base_unit:remove()
end

-- 4. 重新初始化所有模块
Game.init(player)      -- 清理timer/状态/重建基地/开始波次
UI.init(player)        -- 重置HUD/隐藏面板/重绑回调
Extras.register_attack_events()  -- 重新注册伤害事件
```

> ⚠️ 注意：`Extras.register_attack_events()` 未做防重复检查（`y3.game:event` 叠加），目前因 restart 仅在 init 后调用一次所以未触发问题，但若后续需热重载支持应加 `_registered` 标记。
