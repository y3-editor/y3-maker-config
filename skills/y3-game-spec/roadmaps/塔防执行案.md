## Phase 1：物编准备 + 场景模板匹配

> 使用技能：`y3-obj-edit` + `y3-terrain-template`

### 场景模板匹配

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 读取策划案 §3.5 | 提取：塔防类型、4 建造区域、14 航点路径、1 刷怪点、1 基地 |
| 2 | 扫描模板库 | `.codemaker/skills/y3-terrain-template/library/` |
| 3 | 匹配判定 | 模板 `easy-tower-defense-terrain`：塔防 、32×32、4 建造区 、14 航点 |
| 4 | 用户确认后导入 | 5 步 MCP 导入完成 |
| 5 | 实际尺寸 | **保留模板原始 32×32（实测 33×33），未执行 resize** |

> **与策划案差异**：策划案标称 128×128，本执行案落地为 32×32（沿用模板）。如后续需扩容，调用 `resize_terrain` 即可，路径航点需重新调整。

### 🔒 模板使用强制约束（复现工程时必须遵守）

> 复现本工程时，**禁止 AI 自由发挥**地形/装饰物/路径布局，**必须严格使用 `easy-tower-defense-terrain` 模板的原始配置**。

| 约束项 | 强制要求 | 禁止行为 |
|-------|---------|---------|
| **模板来源** | 必须使用 `.codemaker/skills/y3-terrain-template/library/easy-tower-defense-terrain/` | 禁用其他模板，禁用 `y3-gen-terrain-from-image` 重生成 |
| **导入方式** | 必须通过 `y3-terrain-template` skill 的 `import_terrain_template.py` 一次性导入完整 9 文件 | 禁止只导入部分文件，禁止 AI 手动摆放装饰物 |
| **装饰物** | 完全沿用模板 `decorationdata.data` + `editor_decoration.zip`（来源 `easyTD` 关卡的实际布局），保证装饰物位置/数量/类型与原工程一致 | 禁止 AI 调用 `entity_create_block` 增删任何装饰物 |
| **资源摆件** | 完全沿用模板 `resourceobjectdata.data` | 禁止增删资源摆件 |
| **地形/纹理/植被** | 完全沿用模板 `terrain.json` / `texture.json` / `terrainedit.json` / `foliage.json` / `texturefoliage.json` | 禁止 AI 调用任何 `terrain_*` MCP 修改 |
| **路径航点** | 必须沿用模板 14 航点（编辑器 `res_id=10000`），航点坐标由模板的 `editor_decoration.zip` 还原 | 禁止 AI 用 `add_point_path` 重新摆放，禁止微调任何航点坐标 |
| **刷怪点/基地坐标** | 取路径首/末航点（`Config.PATH_POINTS[1]` / `[#PATH_POINTS]`），与模板自动对齐 | 禁止写死其它坐标 |
| **地图尺寸** | 必须保留 32×32（实测 33×33），与模板原尺寸一致 | 禁止 `resize_terrain` 改尺寸（改了路径会失配） |

> **复现 SOP**：
> 1. `python .codemaker/skills/y3-terrain-template/scripts/import_terrain_template.py easy-tower-defense-terrain EntryMap`
> 2. MCP `restart_editor` 让模板生效
> 3. **直接进入 Phase 1 物编生成**，不做任何地形/装饰物/路径调整
>
> 如此可保证装饰物布局、路径走向 100% 与原工程一致，消除"随机种子"与"AI 重摆"两个不确定因素。

## 📖 附录：关键实现参考

### A. 建造技能架构

**选型决策**：使用 **真正的建造技能 `ability_cast_type=4`**（每塔一个独立建造技能），由引擎直接提供建造指示器与碰撞检查。

> **修正历史**：早期评估认为 Y3 建造技能（cast_type=4）"不可用"，需要用 cast_type=1 模拟。实际验证后发现配置正确即可工作（参见根规则「编辑器预制系统信任原则」），已迁移到原生方案。

**技能 ID 映射**：

| 建造技能 ID | 产出塔 ID | 产出塔名 |
|------------|----------|---------|
| `700001`    | 500001 | 箭塔 |
| `134236443` | 500002 | 魔法塔 |
| `134283151` | 500003 | 炮塔 |
| `134231230` | 500004 | 冰塔 |

**物编配置要点**：

1. 4 个独立建造技能，`ability_cast_type=4`，`build_list=[<塔ID>]`
2. 基地（500010）`common_ability_list = [[700001,1],[134236443,1],[134283151,1],[134231230,1]]`
3. 普攻技能 `50000101~50000401`（cast_type=1，作为 `simple_common_atk` 普攻数据载体），分别挂在对应塔身上承载弹道/受击特效

**关键物编字段对照**：

| 字段 | 建造技能（700001 等） | 普攻技能（50000101 等） |
|------|----------------------|------------------------|
| `ability_cast_type` | `4` 建造 | `1` 普通 |
| `sight_type` | `4`（建造指示器） | `0`（无指示器） |
| `is_immediate` | `false`（需玩家选位置） | `true`（被动立即生效） |
| `ability_build_subtype` | `1` | — |
| `build_list` | `[<塔单位ID>]` | 空 |
| `simple_common_atk` | — | 配在**单位**上，不在 ability 上 |

**Lua 监听**（`td_game.lua` 的 `M.register_events`）：

```lua
y3.game:event('技能-建造完成', function(trg, data)
    local tower = data.build_unit
    if not tower then return end
    local key = tower:get_key()
    local data = Config.TOWER_DATA[key]
    if not data then return end
    if State.gold < data.cost then
        tower:remove() return
    end
    State.spend_gold(data.cost)
    Build.tower_levels[tower] = 1
    tower:set_attr('物理攻击', data.damage, '基础')
end)
```

**关键事件**：

| 事件 | 触发时机 | 用途 |
|------|---------|------|

### C. `set_attr` / `get_attr` 正确用法

```lua
-- set_attr(属性名, 值, 属性类型)
tower:set_attr('物理攻击', 50, '基础')          -- 覆盖物编基础攻击力（推荐）
enemy:set_attr('移动速度', 200, '基础')          -- 覆盖物编基础移速
local atk = tower:get_attr('物理攻击', '基础')   -- 获取

-- 属性类型：'基础' / '基础加成' / '增益' / '增益加成' / '最终加成'

-- 错误：'atk_base' / 'ATTACK_PHY' 不是合法 UnitAttr 键
tower:set_attr('atk_base', 50, '基础')  -- 不会报错但属性不改变！
```

**`set_attr` vs `add_attr`**：必须用 `set_attr` 而非 `add_attr`，因为物编基础值只是模板，需要用配置表值完全覆盖。

> 参考：`y3/object/editable_object/unit.lua:698` `y3_editor_object:set_attr()`、`y3/game/const.lua` UnitAttr/UnitAttrType

### F. 事件幂等性

全局事件 `y3.game:event(...)` 和 UI 事件 `player:event(...)` 多次注册会叠加，必须加标记防重复：
```lua
local _events_registered = false
function register_events(player)
    if _events_registered then return end
    y3.game:event('技能-建造完成', callback)
    y3.ui.get_ui(player, 'btn'):add_fast_event('左键-点击', callback)
    _events_registered = true
end
```

**到达终点检测**（ltimer 轮询，替代不可靠的 `'单位-移动结束'` 事件）：
```lua
local t = y3.ltimer.loop(0.5, function(t)
    if not unit:is_exist() then t:remove() return end
    local pos = unit:get_point()
    if pos and endpoint then
        if pos:get_distance_with(endpoint) <= 80 then
            t:remove()
            -- 扣血 + 移除
        end
    end
end)
```

### H. 初始化与热重载

**入口**（`可重载的代码.lua`）：
```lua
include 'td_config'   -- include 支持热重载（require 不支持）
include 'td_state'
include 'td_build'
include 'td_extras'
include 'td_game'
include 'td_ui'

-- 三层触发机制确保所有场景都能初始化
y3.game:event('游戏-初始化', init_td)              -- 正常启动
y3.ltimer.wait(0.1, function() init_td() end)      -- 编辑器直跑
y3.reload.onAfterReload(function(_, has_reloaded)  -- 热重载
    if has_reloaded then init_td() end
end)
```

**init() 幂等性**（必须4项清理）：
1. 清理旧 timer → `M.timers` 追踪所有 `ltimer`
2. 清理旧单位 → 遍历 `alive_monsters` + 移除 `base_unit`
3. 重置状态 → `State.reset()`
4. 重新获取路径 → `get_road()`（重载后旧引用失效）
