# Y3 引擎能力边界清单（Hard Limits）

> 本文件由 `y3-game-spec` 阶段 1 在**每个 Step 用户作答之后立即查阅**。
> 命中任一 ❌ 边界 = 可行性审查直接判**不通过**，必须更换方案，**禁止"勉强凑活"**。
> ⚠️ 项 = 可做但需明确降级，必须当场告知用户。

---

## 🎯 使用方式（强制）

```
每次 ask_user_question 收到用户回答后：
    │
    ▼
┌─────────────────────────────────────┐
│ 1. 解析用户回答中的关键词/参考游戏    │
│ 2. 对照下方边界清单逐条匹配          │
│ 3. 命中 ❌ → 立即打断，给替代方案    │
│ 3. 命中 ⚠️ → 告知降级影响，让用户拍板│
│ 3. 未命中 → 记录答案，进入下一 Step  │
└─────────────────────────────────────┘
```

匹配方式：
- **显式匹配**：用户提到具体游戏名（"像兽人必须死一样"、"参考 CS"、"做个塞尔达"）→ 查"参考游戏命中表"
- **隐式匹配**：用户描述的玩法特征（"第三人称"、"WASD自由探索 3D 大地图"、"横版跳跃"）→ 查"特征边界表"

## 1. 视角与相机 🎥（Y3 是 2.5D 俯视角引擎）

| 边界 | 等级 | 不支持原因 | 命中典型需求 |
|------|------|-----------|-------------|
| 第三人称跟随相机（角色背后/肩后视角） | ❌ | Y3 无第三人称跟随相机 API，相机自由度不足以做 3D 跟随 | 兽人必须死、黑暗之魂、原神、艾尔登法环 |
| 第一人称视角 | ❌ | 同上，无第一人称相机模式 | CS、Apex、Valorant、Minecraft |
| 横版 2D 卷轴（侧视角） | ❌ | Y3 是俯视角引擎，相机不能侧转 | 空洞骑士、奥日、马里奥、银河战士 |
| 自由 3D 探索相机（玩家自由旋转视角） | ❌ | 相机仅支持俯视角范围调整 | 塞尔达、GTA、原神 |
| 俯视角（默认 90°垂直/45°斜俯） | ✅ | 引擎原生支持 | War3、红警、Dota2、暗黑破坏神 |
| 轻度可调俯视角（可拉远拉近、有限度旋转） | ✅ | 引擎支持 | 流放之路、Hades、英雄无敌 |
| 固定单屏俯视（无相机移动） | ✅ | 引擎支持 | 弓箭传说、吸血鬼幸存者 |

## 2. 操作系统 🎮

| 边界 | 等级 | 说明 |
|------|------|------|
| 鼠标准星精确瞄准射击（FPS 式） | ❌ | 俯视角下鼠标只能指向"地面方向"，不是 FPS 那种 3D 准星 |
| 物理碰撞驱动的平台跳跃 | ❌ | Y3 无横版 2D 物理引擎 |
| WASD + 鼠标视角的 3D 自由移动 | ❌ | 与第三人称相机绑定，命中视角边界 |
| WASD 俯视角移动（角色在俯视角下用键盘走） | ⚠️ | 可做，但需禁用默认 RTS 右键移动；"按键朝向"需自写逻辑 |
| RTS 鼠标点选 + 右键移动（默认） | ✅ | Y3 原生支持，零成本 |
| 鼠标点击放置/选择（卡牌/塔防） | ✅ | UI + 单位创建 API 即可 |

## 3. 物理与碰撞 ⚙️

| 边界 | 等级 | 说明 |
|------|------|------|
| 真实物理：抛物线、弹性碰撞、布料、流体 | ❌ | Y3 没有真实物理引擎 |
| 平台跳跃的精确碰撞（站在边缘要不要掉下去） | ❌ | 无横版物理 |
| 投射物简单弹道（直线/抛物线） | ✅ | 物编投射物原生支持 |
| 单位之间的简单碰撞（不能穿过） | ✅ | 引擎原生支持 |

## 4. 网络与规模 🌐

| 边界 | 等级 | 说明 |
|------|------|------|
| 100v100 大型团战 / MMO 千人同屏 | ❌ | 引擎性能上限远不到，会严重卡顿 |
| 实时跨房间联机（动态匹配/全球房间列表） | ⚠️ | 有限支持，需走 Y3 平台对战机制 |
| 1-12 人房间内同步多人 | ✅ | 原生支持 |
| 单机/本地多人 | ✅ | 原生支持 |

## 5. 资源与表现 🎨

| 边界 | 等级 | 说明 |
|------|------|------|
| **导入自定义资源（模型 / 贴图 / 音效 / 特效 / UI 素材）** | ✅ | **Y3 原生支持自定义资源导入**，可使用自制或第三方资源；同时也可使用 Y3 官方资源市场素材 |
| 极端规格的 3D 模型（超高面数 / 复杂骨骼蒙皮） | ⚠️ | 需符合 Y3 编辑器导入规范（格式 / 面数 / 贴图尺寸限制），超规格资源需精简 |
| 实时光线追踪 / 物理 PBR 渲染 | ❌ | 引擎渲染层无此能力 |
| 屏幕后处理（色调/景深/泛光） | ⚠️ | 部分支持，效果受限 |
| 粒子特效（爆炸/火焰/光环） | ✅ | 原生支持 |
| 单位换装/部件组合 | ⚠️ | 需用多个单位/挂件叠加，非真换装系统 |

## 📋 参考游戏命中表（用户说"像XX一样"时立即查这里）

| 用户原始需求 | 命中边界 | 不通过 / 降级原因 | Y3 可行替代方案 |
|-------------|---------|------------------|----------------|
| 像兽人必须死一样 | ❌ 第三人称 3D | 视角不匹配 | 俯视角塔防 + 英雄可主动操控（类《Bloons TD 6》英雄模式 / 暗黑+塔防混合） |
| 像黑暗之魂/艾尔登法环 | ❌ 第三人称 3D 动作 | 视角+操作不匹配 | 俯视角 ARPG（类《Hades》《暗黑破坏神》） |
| 像 CS / Apex / Valorant | ❌ 第一人称 FPS | 视角+瞄准不匹配 | 俯视角射击（类《Hotline Miami》《撤离题材俯视角》） |
| 像空洞骑士 / 奥日 / 马里奥 | ❌ 横版卷轴 | 视角不支持，无横版物理 | **建议换品类**，Y3 不适合横版 |
| 像塞尔达旷野之息 | ❌ 自由 3D 探索 | 视角不匹配 | 俯视角开放世界（类《十字军之王》《环世界》风格） |
| 像 GTA | ❌ 自由 3D 大地图 + 真实物理 | 多重边界 | 俯视角动作冒险（类《GTA1/2 俯视角时代》） |
| 像原神 | ❌ 第三人称 3D 开放世界 | 视角不匹配 | 俯视角 RPG，仅保留"角色切换 + 元素反应"机制 |
| 像 LOL / Dota2 | ✅ 俯视角 MOBA | 完全可行 | 直接做 |
| 像 War3 / 星际 | ✅ 俯视角 RTS | 完全可行 | 直接做 |
| 像吸血鬼幸存者 / 弓箭传说 | ✅ 俯视角 Roguelike | 完全可行 | 直接做 |
| 像植物大战僵尸 / Bloons TD | ✅ 俯视角塔防 | 完全可行 | 直接做 |
| 像炉石传说 / 万智牌 | ✅ 卡牌 UI 驱动 | 完全可行 | 直接做 |
| 像魔兽世界 / FF14 / 剑网3 | ❌ 持久世界 MMO | 场次形态不匹配（Y3 是单局制） | 改为单局制副本 RPG（类《暗黑破坏神》《火炬之光》） |
| 像巫师3 / 上古卷轴 | ⚠️ 长流程开放世界 | 规模和单局形态不匹配 | 拆成多局闯关式 RPG，每局聚焦一个主线章节 |

## 📋 特征边界表（用户描述功能特征时查这里）

| 用户描述包含 | 命中边界 | 处理 |
|-------------|---------|------|
| "第三人称"、"角色背后视角"、"摄像机跟着角色" | ❌ 视角#1 | 改俯视角 |
| "第一人称"、"FPS"、"第一视角" | ❌ 视角#2 | 改俯视角射击 |
| "横版"、"侧视角"、"2D 平台跳跃" | ❌ 视角#3 | 换品类 |
| "自由旋转视角"、"3D 开放世界"、"自由探索 3D" | ❌ 视角#4 | 改俯视角开放 |
| "鼠标准星瞄准"、"打哪指哪" | ❌ 操作#1 | 改"指向地面方向"或"自动锁定最近敌人" |
| "WASD 跳跃 + 物理跳台" | ❌ 物理#2 | 不做平台跳跃，改俯视角动作 |
| "100v100"、"千人战场"、"MMO" | ❌ 规模#1 | 缩减到 1-12 人或 PvE 大量小怪 |
| "实时光追"、"次世代渲染" | ❌ 渲染#2 | 删除该需求 |
| "随便导入任意 3D 模型" | ✅ 资源#1 | Y3 支持自定义资源导入，仅需符合编辑器规范（格式/面数等），告知规范即可 |
| "持久世界"、"跨服 MMO"、"千人同服在线" | ❌ 场次#1 | Y3 是单局制引擎，建议改为"1-12 人房间对战"或"单机闯关" |
| "几十小时主线开放世界"、"自由大地图持久存档" | ⚠️ 场次#2 | 拆成多局闯关式结构，每局承载一段剧情 |

## 🔧 边界触发后的标准处理流程

### ❌ 命中硬边界

使用 `ask_user_question` 提问（**必须打断当前 Step**）：

```
问题: ⛔ 检测到你的需求触碰 Y3 引擎边界，无法实现：

  ❌ [具体需求] — 原因：[边界说明]

  Y3 是 2.5D 俯视角引擎，建议改为以下方向：

选项:
  - [替代方案 A]（推荐，最接近原始需求）
  - [替代方案 B]（保留部分核心体验）
  - [替代方案 C]（完全换方向）
  - 我坚持原始需求，不在 Y3 上做了（终止流程）
```

### ⚠️ 命中降级项

使用 `ask_user_question` 提问（**可继续 Step，但必须告知**）：

```
问题: ⚠️ 你的需求在 Y3 上能实现，但需要降级：

  ⚠️ [具体需求] — 降级说明：[原始 vs 降级版的差异]

选项:
  - 接受降级方案（推荐）
  - 保留原始需求并标注风险（写入策划案第 10 章）
  - 去掉该需求
```

## 🎯 与 game-design-guide.md 的衔接

### 在 Step 1 之后必须执行（前置审查）

如果用户在 Step 1 选了**[其他]**并描述了具体游戏名/参考游戏，立即查"参考游戏命中表"。
命中 ❌ 必须打断，否则进入 Step 2。

### 在 Step 2~8 每步之后必须执行（实时审查）

每次收集到用户回答后，立即查"特征边界表"+"参考游戏命中表"。
命中即处理，禁止积累到 Step 9。

### Step 9 的角色重新定位

Step 9 不再做"逐项扫描全部需求"，仅做：
1. **跨 Step 冲突检测**：例如 Step 4 选了"控制效果" / Step 5 又选了"陷阱不要特效" → 互斥
2. **最终汇总确认**：列出之前所有 ❌/⚠️ 的处理结果，让用户最后过一遍
