# Y3 地形邻格约束规则

> 本文档从源码 `imp_terrain_edit.py` 和 `edit_terrian_cliff_state.py` 整理而来。
> 描述 Y3 编辑器刷地形时的自动联带修改规则，用于指导 skill 的分层写入顺序。

---

## 一、地形格子的基本参数

每个格子（cell）由以下参数定义：

| 参数 | 类型 | 说明 |
|------|------|------|
| `terrain_height` | int | 地形高度，范围约 `[-38, 215]`，单位为格 |
| `flat` | int | 斜坡/悬崖类型（见下表） |
| `has_water` | bool | 是否有水 |
| `water_height` | int | 水体高度（相对地形的水深，0~3 格） |
| `water_type` | int | 水类型（深水/浅水/平面水） |
| `mesh_type` | int | 网格类型（悬崖/斜坡几何体） |
| `cliff_mat` | int | 悬崖材质 ID |

### 特殊高度常量

| 常量 | 值 | 说明 |
|------|-----|------|
| `CrackTerrainHeight` | -40 | 裂缝格（crack），完全不可通行的特殊格 |
| `MinTerrainHeight` | -38 | 正常地形最低高度 |

---

## 二、`flat` 值（平面/斜坡类型）

`flat` 是最核心的地形约束参数，决定格子是普通地形还是斜坡，以及斜坡的方向：

| `flat` 值 | 含义 | 等效高度处理 |
|-----------|------|-------------|
| `0` | 普通地面（cliff/water/ground 均使用） | 无调整 |
| `1` | 横向斜坡（X 轴方向，左-右过渡） | 等效 `height - 1`（斜坡格在约束计算中高度减 1）|
| `2` | 纵向斜坡（Z 轴方向，前-后过渡） | 等效 `height - 1` |
| `3` | 通用斜坡（可双向，自动匹配方向） | 等效 `height - 1` |

> **关键**：斜坡格（flat != 0）在邻格约束计算时，其有效高度为 `terrain_height - 1`，这是因为斜坡格跨越了两个高度层级。

---

## 三、核心联带修改机制：`place_region` 波纹传播

**源码位置**：`place_region` 函数（第 1289 行）— **这是所有地形修改后联带修改的核心入口**

### 传播原理（BFS 波纹扩散）

`place_region` 接受一个"意图修改区域" `region`（dict），然后向外扩散验证，直到收敛：

```
while 未收敛:
    edge = 上一圈格子的外围 8 邻格（排除已处理过的格）

    for edge 中每个格:
        以 region 内已确定的格为参考，计算本格的合法高度范围
        [min_terrain_height, max_terrain_height] 和 [min_water_surface, max_water_surface]
        对多个参考点取最严格交集

    用 apply_value_range 将当前实际高度 clamp 到合法范围
    如果 clamp 后高度发生变化 → ok = False（继续下一轮扩散）

直到外圈所有格高度都无需修改为止（收敛）
```

**关键结论**：每次写入一个地形区域，引擎会向外**无限扩散**验证，直到所有邻格都满足约束。影响范围可能远超写入区域本身。

---

### 规则 1：水体与陆地边界（`cal_value_range` 计算约束）

```
规则 1a：有水格 → 相邻无水格
  无水格的地形高度 ≥ (有水格的 terrain_height + water_height + 1)
  即：陆地必须严格高于相邻水面至少 1 格

规则 1b：无水格 → 相邻有水格
  若当前水面 > 无水格地形高度：
    水面被压低到 ≤ 无水格地形高度
  即：水体不能高于相邻陆地
```

斜坡格在参与约束计算时，高度等效为 `terrain_height - 1`（跨越两个层级）。

**实际影响**：
- 写一个水格 → 外圈所有无水格地形被抬高到水面以上
- 抬高外圈格 → 可能触发更外圈的约束调整（继续扩散）

---

### 规则 2：斜坡（flat）的放置顺序

**源码位置**：`place_region` 第 1407~1438 行

引擎在写完所有地形高度后，**再进行三轮斜坡验证**（`calculate_valid_flat`），顺序固定：

```
第 1 轮：放置 flat=3（角落斜坡）
  角落斜坡不依赖垂直/水平斜坡，先放可以锚定角落

第 2 轮：放置 flat=1（横向）和 flat=2（纵向）
  依赖两侧高差恰好为 2

第 3 轮：再次放置 flat=3（角落斜坡）
  部分角落斜坡需要邻近存在垂直斜坡后才能激活，所以要放第二次
```

**激活条件（flat=1，横向斜坡）**：
```
左高 - 右高 = 2（或 右-左=2）
且上下邻格与矮侧等高，或也是斜坡
且 flat=1 的格子的 ±Z 方向不存在 flat=2 的格子
→ 斜坡高度 = (左高 + 右高) / 2
```

**激活条件（flat=2，纵向斜坡）**：
```
上高 - 下高 = 2（或 下-上=2）
且左右邻格与矮侧等高，或也是斜坡
且 flat=2 的格子的 ±X 方向不存在 flat=1 的格子
→ 斜坡高度 = (上高 + 下高) / 2
```

### 规则 3：斜坡格的高度奇偶性处理

**源码位置**：`place_region` 第 1392 行

```python
if terrain_height & 1:   # 奇数高度格 → 斜坡候选
    terrain_height -= 1  # 先降为偶数（低一级）
    if is_water:
        water_height += 1  # 水深补偿
```

**含义**：高度为奇数的格子在写入前会被先降 1，然后在后续斜坡验证中重新判断是否激活为斜坡。这意味着：
- **斜坡格的最终高度不由你直接控制**，而是由引擎根据邻格推算
- 主动在 CSV 中写奇数高度格，会被引擎先降为偶数，再可能升回去（作为斜坡的低侧高度）

---

### 规则 4：水体类型蔓延（`spread_water_type`）

**源码位置**：`spread_water_type` 函数

刷水后，引擎会 DFS 扩散水体类型到所有相连的水格：

```
从 region 边缘的水格出发，DFS 搜索所有与其相连且水类型不同的相邻水格
→ 将这些格子的水类型统一为起始格的水类型
```

**影响**：如果你写入一块新水体，它会感染相邻的旧水体，把旧水体的水类型（深水/浅水）改成新写入的类型。

---

### 规则 5：裂缝格（crack）约束

```
is_crack(gx, gz) ↔ terrain_height == -40

裂缝格规则：
- 不可对裂缝格执行悬崖抬高/降低操作（`do_tile_exchange` 中直接 return）
- 裂缝格周围 8 格不能放置植被/摆件（会被清零）
- 裂缝格本身不存在水体
- 在 cal_value_range 中：邻格是裂缝时，若参考格有 detail_height 则设最小高度；否则直接跳过约束
```

---

### 规则 6：水体刷写的自动围墙（`place_water`）

刷水时，引擎对水域外圈 1 格执行"围墙"操作：

```
围墙格高度 = max(原高度, 水底高度 + 水深 + 1)
```

即外圈陆地格**只会被抬高，不会被降低**。然后整体进入 `place_region` 进行 BFS 传播。

## 四、分层写入最优顺序（基于以上规则推导）

| 层次 | 地形类型 | 原因 |
|------|---------|------|
| **第 1 层** | `cliff`（悬崖，flat=0，高度很高） | 悬崖是最强约束，确定后水体和斜坡的位置才能确定 |
| **第 2 层** | `water`（水体） | 水体写入会修改外圈 1 格高度（围墙），必须在普通地形前确定 |
| **第 3 层** | `slope`（斜坡，flat=1/2/3） | 斜坡依赖两侧高差恰好为 2，需在悬崖和水体定型后才能激活 |
| **第 4 层** | `ground`（普通地面） | 最"软"的地形，由前三层约束自然填充，高度最灵活 |
| **第 5 层** | 摆件/植被 | 与地形约束无关，最后放置 |

> ⚠️ **注意**：以上顺序待实际测试后可能需要调整。特别是 cliff 和 water 的顺序，如果水体与悬崖相邻，可能需要先刷悬崖确保围墙高度正确。

## 五、实际写入建议

### 5.1 写入时尽量从高度值大到小

由于水体写入会自动将外圈抬高，**先写高地形（cliff/高 ground），再写水体**，避免水体把尚未写入的低地形格子意外抬高。

### 5.2 斜坡不需要主动写，让引擎自动生成

斜坡（flat=1/2/3）是引擎根据邻格高差**自动计算**的，不需要在点阵中主动刷写斜坡。  
**最佳做法**：只在点阵中标记 cliff/water/ground 的高度，让引擎自动在高差为 2 的边界处生成斜坡。

### 5.3 迭代修正对斜坡格意义不大

由于斜坡是自动生成的，如果迭代读回发现斜坡格与目标不一致，通常是因为邻格高差不满足条件，强行写入斜坡 flat 值无法生效。应检查邻格高差是否为 2。

### 5.4 裂缝格的处理

裂缝格（`terrain_height == -40`）是通过**专用接口主动刷入**的地形类型，不是引擎自动产生的副作用。

> ⚠️ **术语说明**：Y3 中 `cliff` 特指裂缝（地形空洞），**不是悬崖或高山**。图片识别时，山峰图案是 ground 局部高地（高出周围 +5），不是 cliff。

在生成地形点阵时，裂缝区域在 `terrain_grid.csv` 中标记为 `cliff` 类型，Pass 1 写地形时对 `cliff` 格子通过 MCP 裂缝专用接口（`terrain_set_crack_block`）写入，**不要用普通的地形高度写入接口写 `-40`**。
