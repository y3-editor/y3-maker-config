## 模型选择工作流（v3）

```
decoration_input.json                  texture_grid.csv
  (AI 标注 tree_cluster /               (Round 1 产出)
   mountain_chain + 方位)                    │
         │                                   │
         ▼                                   ▼
  decoration_postprocess.py ◄── texture_group_catalog.json
         │
         │  每个采样点: (x,z) → 查纹理ID → 查纹理组 → 选模型池 → 随机抽取
         │
         ▼
  decoration_entities.json
  (每个实体已填入纹理匹配的 model_id)
```

### Fallback 行为

- 无 `--texture-grid` 参数 → 所有点使用 `grassland` 组模型
- 纹理 ID 不在任何组中 → 使用 `fallback_group`（默认 `grassland`）
- 纹理组 `trees` 为空（如 `rock`/`road`） → 跳过该点树木放置

## AI 使用指南

### 装饰物放置（entity_create_block）
```
decoration_entities.json 由脚本自动生成，格式：
[
  {
    "type": 16777216,       // RESOURCE_MODEL = 2^24
    "pos": [x, y, z],      // 世界坐标
    "model_id": <auto>,     // 由纹理组自动匹配
    "yaw": <random>,        // 随机旋转
    "pitch": 0,
    "roll": 0,
    "scale": [sx, sy, sz],  // 山脉自动 scale，树木 [1,1,1]
    "stick_to_ground": true
  }
]
```

### 植被绘制（vegetation_draw_block）
```
terrain_vegetation_draw_block({
    "cells": [{
        "x": x, "z": z,
        "vegetation_type": vegetation_type,
        "density": 100
    }]
})
```

### 道路密铺（road 类型特殊处理）

道路不是一个大模型，而是由**大量小正方形模型密铺**拼成的：

```
道路铺设：沿中心线锚点方向，每格放一个 model_id=205148
┌─────┬─────┬─────┬─────┬─────┐
│205  │205  │205  │205  │205  │  ← 每格一个方块模型
│148  │148  │148  │148  │148  │  ← stick_to_ground=true
├─────┼─────┼─────┼─────┼─────┤
│205  │205  │205  │205  │205  │  ← 宽度方向也密铺
│148  │148  │148  │148  │148  │  ← 形成连续道路
└─────┴─────┴─────┴─────┴─────┘
```
