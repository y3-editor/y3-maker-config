# Round 2 AI Prompt：装饰物识别

> **本轮目标**：判断地图风格 → 识别桥梁、树木、山脉位置 → 生成装饰物数据。
> **本轮产出**：decoration_entities.json (由 decoration_postprocess.py 生成)

---

## AI 输入

1. **原始地图图片** — 主要参考
2. **water_map.txt** — 精简水域字符地图（由 `gen_water_map.py` 从 terrain_grid.csv 生成）。格式：`W`=水域 `.`=陆地，行号=z 列号=x。**桥梁坐标必须落在 `W` 格子上。**
3. **water_mask_grid.npy** — 水域格子标记（Round 1 产出，供脚本使用）
4. **continent_map_grid.npy** — 大陆编号图（Round 1 产出）

---

## Step 2.0: 地图风格判断

### 任务

根据原始地图图片的视觉特征，判断地图整体风格，选择一个 `style` 值。后处理脚本会根据此风格自动选取风格匹配的树木和岩石模型。

### 风格选择表

| style_id | 中文名 | 适用场景 |
|----------|--------|---------|
| `temperate` | 温带写实 | 温带草原/平原，绿色阔叶林，大多数奇幻/策略/三国类地图 |
| `tropical` | 热带写实 | 热带雨林/海岛，棕榈树和阔叶热带植物，海岛、丛林地图 |
| `boreal` | 寒带写实 | 寒带/雪原，针叶林（松树/云杉），雪山、北方、冬季地图 |
| `desert` | 荒漠 | 沙漠/荒漠，枯树、仙人掌、干旱植物，沙漠、西部地图 |
| `oriental` | 国风仙侠 | 中国风/仙侠，桃树、银杏、古典园林，三国、武侠、仙侠地图 |
| `fantasy` | 卡通魔幻 | 卡通/Q版/魔幻，色彩鲜艳的树木和岩石，休闲、萌系、奇幻冒险地图 |
| `dark` | 末世暗黑 | 末世/暗黑/恐怖，枯树、黑色岩石，末日生存、暗黑地牢地图 |

### 判断示例

- 地图以**沙黄色**为主，有大片沙丘 → `desert`
- 地图有**雪白色区域**，针叶树图案 → `boreal`
- 地图以**翠绿色**为主，有常见的草原/森林 → `temperate`
- 地图画风**卡通**，颜色鲜艳饱和 → `fantasy`
- 地图有**桃花**、**飘落花瓣**、古风建筑 → `oriental`

### 输出

选定一个 `style` 值（从上表 style_id 列选取），在后续 Step 2d 的 `decoration_input.json` 中作为 `"style"` 字段输出。

> **注意**：如果地图风格不明确或不属于上述任何一类，默认使用 `temperate`。

## Step 2a: 桥梁识别

### 任务

从原始地图图片中识别桥梁位置。

### 桥梁特征

1. **跨越水域**：桥梁必须横跨水域区域（蓝色河流）
2. **连接两个大陆**：桥两端分别连接不同大陆的陆地
3. **视觉特征**：通常呈现为横跨河流的条状结构，颜色与周围水域不同（棕色/黄色/灰色绳索/木板结构）

### 输出格式

```json
[
  { "x": 30, "z": 15, "yaw": 90, "note": "左上方河流上的木桥" },
  { "x": 42, "z": 28, "yaw": 0, "note": "中央河流上的绳桥" }
]
```

> **⚠️ 坐标精度要求**：桥梁坐标 `(x, z)` **必须落在 `water_map.txt` 中标记为 `W` 的格子上**。请对照 `water_map.txt` 验证每座桥的坐标——如果 `water_map.txt` 中 `(x, z)` 处不是 `W`，请调整坐标到最近的 `W` 格子。
> **yaw 字段（可选）**：桥梁朝向角度，0=朝北（南北向），90=朝东（东西向），顺时针为正。脚本会自动推算最佳 yaw，AI 给的 yaw 仅作为参考 fallback。
> 保存为 `bridge_positions.json`。

---

## Step 2b: 树木识别

### 任务

从原始地图图片中识别树丛区域。

### 树木特征

1. **绿色团状图案**：区别于大陆底色的明显绿色团块
2. **有特定形状**：通常呈圆形或椭圆形的树冠图案
3. **分布在陆地上**：不在水域中

### 输出格式

AI 需要标注树丛区域的范围（网格坐标矩形），脚本会在区域内用泊松圆盘采样自然散布树木（自动过滤水域上的点）：

```json
[
  { "x_min": 20, "z_min": 10, "x_max": 25, "z_max": 14, "note": "左上方小树林" },
  { "x_min": 40, "z_min": 35, "x_max": 48, "z_max": 42, "note": "右下方大树林" }
]
```

> **精度要求**：网格坐标，覆盖图中可见树丛区域即可。
> 保存为 `tree_regions.json`。

---

## Step 2c: 山脉识别

### 任务

从原始地图图片中识别山脉/高地区域。山脉会放置山峰和岩石装饰物。

### 山脉特征

1. **三角形山峰符号**：手绘地图中常见的山脉图案
2. **灰色/深色岩石区域**：与周围草地/泥土颜色不同
3. **位于陆地上**：不在水域中

### 输出格式

```json
[
  { "x_min": 5, "z_min": 5, "x_max": 15, "z_max": 12, "note": "左上山脉区" },
  { "x_min": 30, "z_min": 40, "x_max": 45, "z_max": 50, "note": "中下方山脉" }
]
```

> **精度要求**：网格坐标矩形，覆盖图中可见山脉区域即可。
> 保存为 `mountain_regions.json`。

---

## Step 2d: 调用脚本生成装饰物实体

AI 将 Step 2.0 的风格判断和 Step 2a/2b/2c 的结果合并为一个 JSON 输入文件：

```json
{
  "style": "temperate",
  "trees": [{"x_min":20,"z_min":10,"x_max":25,"z_max":14}, ...],
  "bridges": [{"x":33,"z":10,"yaw":90}, ...],
  "mountains": [{"x_min":5,"z_min":5,"x_max":15,"z_max":12}, ...]
}
```

> **`style` 字段必填**：使用 Step 2.0 中判断的风格 ID。脚本会根据此字段自动选取匹配的树木和岩石模型。

保存为 `decoration_input.json`，然后调用后处理脚本：

```bash
python scripts/decoration_postprocess.py \
  decoration_input.json \
  water_mask_grid.npy \
  decoration_entities.json
```

> 地图参数自动从同目录下的 `map_info.json` 读取，AI 无需传入。

脚本自动完成：
- 树木泊松圆盘采样（自然散布，非矩形填充）+ 水域过滤 + 随机 yaw/scale
- 桥梁水域吸附 + yaw 自动推算（垂直于河流方向）
- 山石泊松散布 + 混用山峰/岩石模型 + 水域过滤
- 输出 `decoration_entities.json`，可直接传给 `entity_create_block` MCP 调用

> ⛔ **禁止 AI 自行编写脚本生成装饰物**，必须调用 `scripts/decoration_postprocess.py`
